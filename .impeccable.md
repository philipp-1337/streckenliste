# Impeccable Design Context

## Design Context

### Users

Professional and licensed hunters (Jäger) managing official game records (Strecke) for a shared hunting district (Jagdrevier). They use the app both in the field on mobile and at home/club meetings on desktop. Multiple users share the same district data. The app handles legally relevant documentation — entries must be accurate and trustworthy.

### Brand Personality

Modern · Sleek · Outdoorsy

The app should feel like a well-crafted professional tool built for people who take their craft seriously — not a consumer app, not enterprise bloat. Think of a precision instrument: refined, purposeful, nature-rooted.

### Emotional Goal: Efficient & Frictionless

Users want to log a kill quickly and move on. Especially in the field. Every extra tap, unclear label, or sluggish interaction is a failure. The app should feel effortless and fast — data entry should be smooth, feedback immediate, navigation obvious.

### Aesthetic Direction

- **Color:** Deep hunting green (`green-800` #166534) as the anchor, lighter greens (`green-500`, `green-50`) for hierarchy and backgrounds. No blue-dominant palettes.
- **Effects:** Glassmorphism for floating UI elements (nav, action buttons) — feels modern and premium without being distracting.
- **Theme:** Light mode primary (green-50 backgrounds), dark nav/header contrast. Dark mode not required but logo supports it.
- **Typography:** System fonts (clean, legible, no decorative fonts). Hierarchy through weight and size, not ornamentation.
- **Icons:** Lucide icons — consistent, clean, no emoji or decorative glyphs.
- **Motion:** Subtle spring-curve animations (`cubic-bezier(0.34, 1.56, 0.64, 1)`), minimal and purposeful. No flashy transitions.

### Anti-References

- **No generic SaaS dashboard:** Avoid blue-heavy palettes, dense chart grids, enterprise sidebar nav
- **No hunting store / e-commerce:** No camouflage patterns, trophy aesthetics, product catalog feel
- **No gamified apps:** No streaks, badges, achievement toasts, point systems — this is official documentation

### Design Principles

1. **Speed over flourish.** Every UI decision should reduce friction for data entry. Interactions must feel instant.
2. **Nature-rooted precision.** Green palette, clean lines, no noise — like a quality piece of field equipment.
3. **Mobile-first, field-ready.** Touch targets ≥48px, safe area awareness, offline-capable. Assume the user is standing outside.
4. **Data is the hero.** The table and form are core. Supporting chrome (nav, filters, actions) should recede — the records should dominate.
5. **Professional, not playful.** Official records require a tone of quiet authority. No whimsy, no gamification.

## Tech Stack

- React 19 + TypeScript, Vite, Tailwind CSS 4
- Firebase (Firestore + Auth)
- PWA (vite-plugin-pwa), installable standalone
- Lucide React icons, React Hook Form + Zod validation, Sonner toasts
- No CSS-in-JS, pure Tailwind utilities + custom plugin tokens

## Audit Findings (2026-03-25)

### Critical Issues

1. **Off-palette blue** (`EintragForm.tsx:246`) — `bg-blue-50` info box breaks the all-green palette. Design tokens specify no blue.
2. **Emoji in professional UI** (`EintragForm.tsx:247`) — `📋` emoji in a form heading. Violates "Professional, not playful" + design token rule (Lucide icons only, no emoji).
3. **Cancel button hierarchy** (`EintragForm.tsx:373`) — Solid `bg-gray-500` cancel button competes visually with the primary submit. Secondary actions should use ghost/outline style.
4. **Login page: no branding** (`Login.tsx:54`) — Login card shows plain "Login" text with no logo, app name, or visual identity. Poor first impression.
5. **Login page: hand-rolled spinner** (`Login.tsx:45`) — Loading state uses a custom CSS spinner instead of the project's `<Spinner>` component. Inconsistency.

### Minor Issues

6. **Input focus styles missing** (`EintragForm.tsx`) — Form inputs have no focus ring. Login inputs have `focus:ring-green-500` but EintragForm inputs don't. Accessibility gap + visual inconsistency.
7. **Input border inconsistency** — Login uses `border-gray-300`; EintragForm inputs use plain `border` (browser default color).
8. **Login label color** (`Login.tsx:57,77`) — Uses `text-gray-700`; only form in the app using gray for labels.
9. **Result counter color** (`ActionButtons.tsx:163`) — `text-gray-600` for `X / Y Treffer`; should use green palette.
10. **StatistikPanel card background** — `bg-green-100` cards differ from white cards used elsewhere. **Intentional**: green-100 on the stats page provides visual grouping and stays on-palette. No fix needed.

### Applied Fixes

- ✅ EintragForm info box: `bg-blue-50` → `bg-green-50 border border-green-200`, emoji removed
- ✅ EintragForm Cancel button: solid gray → ghost/outline style
- ✅ EintragForm inputs: added `border-gray-300` + `focus:outline-none focus:ring-2 focus:ring-green-500/30 focus:border-green-500`
- ✅ Login: replaced hand-rolled spinner with `<Spinner>`, added app branding (logo + name)
- ✅ Login labels: `text-gray-700` → consistent with rest of app
- ✅ ActionButtons result counter: `text-gray-600` → `text-green-900/50`
- ✅ Form buttons: `rounded-lg` → `rounded-xl` (EintragForm + Login, per design token)
- ✅ Login submit button: `py-2` → `py-3 min-h-[48px]` (meets ≥48px touch target)
- ✅ Login inputs: added `focus:outline-none`, normalized ring to `ring-green-500/30`
- ✅ FilterPanel: all 4 fields got `border-gray-300` + focus styles + `aria-label`
- ✅ JagdjahrSelect mobile icon: `w-10 h-10` → `w-12 h-12` (40px → 48px touch target)

## Design Tokens

- **Primary:** `green-800` (#166534) — headers, active states
- **Accent:** `green-500` / `green-700` — interactive elements
- **Background:** `green-50` (#f0fdf4) — page, iOS overscroll
- **Surface:** white — cards, table rows
- **Error/Delete:** `red-500` / `red-600`
- **Border radius:** `rounded-xl` buttons, `rounded-lg` containers, `rounded-pill` (36px) nav
- **Glassmorphism:** `glass-bg` + `glass-shadow` utilities, `backdrop-blur-xl`
- **Safe area spacing:** `safe`, `safe-sm`, `bottom-safe-floating` custom tokens
